When the terminal is the Web

webscript.io

  • HTTP oriented scripting console:
    • Scripts are executed on an HTTP request to the script URL address
    • Scripts return a HTTP message (status code, HTML or JSON data)
  • Possibility to program script execution (cron jobs)

webscript.io

  • Basic script facilities:
    • HTTP requests
    • Persistent storage
    • Sending emails
    • Generating and parsing JSON
    • Cryptographic functions
    • Leases
    • Sending alerts to SMS and email (paid accounts)
  • Advanced facilities:
    • Importing modules from Github
    • Webhooks

webscript.io

  • Scripting language: Lua


Lua is a rich, modern programming language that was designed and implemented to be easily embedded and  sandboxed inside another application.

webscript.io

webscript.io

  • Very quick hands-on Lua (I) :
    • Variable declaration: myVariable = 24
    • Multiple assignment: v1, v2 = 9, "hey"
    • By default all variables are global
    • Variables can be local to certain scopes:
      • local myVariable = 24
      • Function
      • Blocks
    • Dynamic typing (nil, number, string, boolean, associative arrays)

webscript.io

  • Very quick  hands-on Lua (II) :

      • Length: # (local counter = #str )
      • Multiline: [[]]
    • Boolean operators ( < , <= , == , >= , > , ~= )
      • and
      • or
      • not

    webscript.io

    • Script invocation:
      • request :
        • query
        • form
        • files
        • headers
        • method
        • body
        • scheme, port, path
        • remote_addr

    webscript.io

    • Script response:
      • A script may respond with 0 or more return values
      • String:
        • Body of the HTTP response message
      • Integer:
        • Status code of the HTTP response message
      • Table:
        • If no String, it is included as JSON in the body of the HTTP response message
        • If there is a String or it is the second table, it is interpreted as Header values

      WEBSCRIPT.IO

      • Persistent storage:
        • storage table
        • Scoped to a specific subdomain
        • Shared to all its scripts
        • Non existent key returns nil
        • Assigning nil to key deletes its value
